Claude Token Compression (Beta) reduces token costs in Claude conversations by compressing tool call results — the large outputs from file reads, 
shell commands, search results, and other tools — before they reach Anthropic models. 

Unlike Agentic compression, it is fully lossless: no information is lost, and no quality configuration is needed.

For Claude Code users, this means **lower costs** when using API billing, and **up to 3× longer sessions** for Pro/Max subscribers: the same token budget goes further.

## What Gets Compressed

Claude Token Compression targets `tool_result` messages in Claude API format. This includes:

- **File contents** — output from Read tool and file system operations
- **Grep and search outputs** — results from code search, ripgrep, and similar tools
- **Shell command output** — stdout/stderr from Bash and terminal commands
- **API responses** — large JSON or text payloads returned by tool calls
- **Database query results** — rows and records returned from tool-executed queries

User messages and assistant turns are not modified, only tool results are compressed.

## How to Enable

In the **Edge Models** section of your console, find the **Claude Token Compression** card and simply select which API keys should use Claude compression.

## Per-Request Override

You can also enable Claude compression directly in the request body using the `compression_model` field, overriding your API key settings. This is useful for testing or for enabling compression on specific requests without changing your API key configuration.

```json
{
  "model": "anthropic/claude-sonnet-4-20250514",
  "messages": [...],
  "compression_model": "claude"
}
```

Available compression models:
- `claude` — tool compression with Claude-specific compressors
- `opencode` — tool compression with OpenCode-specific compressors
- `agentic` — LLMLingua + tool compression + smartcrusher
- `cursor` — tool compression with Cursor-specific compressors
- `customer` — generic customer compression

For the `agentic` model, you can optionally pass a `compression_configuration` object to control compression rate and semantic preservation:

```json
{
  "model": "anthropic/claude-sonnet-4-20250514",
  "messages": [...],
  "compression_model": "agentic",
  "compression_configuration": {
    "rate": 0.7,
    "semantic_preservation_threshold": 60
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `rate` | `float` | Compression rate between 0.0 and 1.0. Defaults to 0.8. |
| `semantic_preservation_threshold` | `int` | Semantic preservation threshold between 0 and 100. |

<Note>
  `compression_configuration` is currently only available for the `agentic` compression model. Other models don't have configuration options yet.
</Note>

These fields are gateway-internal and are never forwarded to providers. When present, they take priority over your API key-level compression settings.

The SDKs also support these fields:

```typescript TypeScript
const response = await edgee.send({
  model: "anthropic/claude-sonnet-4-20250514",
  input: {
    messages: [...],

  },
});

// With agentic compression configuration
const response = await edgee.send({
  model: "anthropic/claude-sonnet-4-20250514",

    messages: [...],
    compression_model: "agentic",
    compression_configuration: {
      rate: 0.7,
      semantic_preservation_threshold: 60,
    },
  },
});
```

```python Python
response = edgee.send(
    model="anthropic/claude-sonnet-4-20250514",

# With agentic compression configuration
response = edgee.send(
    model="anthropic/claude-sonnet-4-20250514",
```

```rust Rust
let input = InputObject::new(messages)

let input = InputObject::new(messages)
    .with_compression_model("agentic")
    .with_compression_configuration(CompressionConfiguration {

    });
```
